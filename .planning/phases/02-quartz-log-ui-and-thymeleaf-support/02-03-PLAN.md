**验收步骤（浏览器访问 http://localhost:8080）：**

1. **登录** — 用管理员账号登录系统（复用 Phase 1 验收用的账号）

2. **导航到调度日志页** — 从侧边栏菜单点击"调度日志"或在地址栏输入 `http://localhost:8080/monitor/jobLog`
   - 期望：页面返回 200，显示调度日志列表页（标题"调度日志列表"）
   - 异常：400/500/NPE/白屏 → 记录错误

3. **检查搜索栏字典下拉框**：
   - 任务组名下拉框：点击展开，应显示 sys_job_group 字典选项（如 DEFAULT、SYSTEM）
   - 执行状态下拉框：点击展开，应显示 sys_common_status 字典选项（如 成功、失败）
   - 期望：两个下拉框均有选项（非空）
   - 异常：下拉框为空 → @dict bean 未生效或字典缓存未加载（检查启动日志 "---初始化数据字典"）

4. **检查工具栏按钮显隐**（shiro:hasPermission 生效）：
   - 管理员账号：应看到"删除""清空""导出"三个按钮
   - 期望：按钮可见（管理员有 monitor:job:remove/export 权限）
   - 异常：按钮全部不可见 → ShiroDialect 未注册或权限配置问题

5. **检查日志列表加载**：
   - 页面加载后，bootstrap-table 应自动发起 AJAX GET /monitor/jobLog/list
   - 期望：表格显示调度日志记录（若有任务执行过）或显示"没有找到匹配的记录"（无日志时）
   - 异常：AJAX 401/403/500 → 检查权限或 Controller 路由

6. **测试删除操作**（若有日志记录）：
   - 勾选一条或多条记录，点击"删除"按钮
   - 期望：确认对话框 → 确认 → 成功提示 → 表格刷新，选中记录消失
   - 异常：POST 405 / DELETE 不匹配 → 检查 removeAllJobLogs 函数

7. **测试清空操作**（谨慎，会清空所有日志）：
   - 点击"清空"按钮
   - 期望：确认对话框 → 确认 → 成功提示 → 表格清空
   - 异常：POST 405 → 检查 cleanJobLogs 函数

8. **测试导出操作**：
   - 点击"导出"按钮
   - 期望：浏览器下载 xlsx 文件，文件名含"调度日志"
   - 异常：403/500 → 检查权限或 ExcelUtil
