已存在路由（line 64-85）：
  @GetMapping() → view() → return "monitor/job/job"    (列表页)
  @GetMapping("/add") → add() → return "monitor/job/add"    (新增表单)
  @GetMapping("/edit/{jobId}") → edit(Long, ModelMap) → return "monitor/job/edit"  (编辑表单)
  @GetMapping("/{jobId}") → getInfo(Long) → @ResponseBody AjaxResult  (详情API — 这是问题所在)

Spring MVC 匹配规则：当 URL 为 /monitor/job/job 时，/@GetMapping() 只匹配 /monitor/job (base path)，剩余的 /job 段会继续匹配。由于 /{jobId} 是单段路径变量模式，Spring MVC 会将 /job 视为 /{jobId} 的匹配（jobId="job"），而不会回退到空的 @GetMapping() 处理器。

修复策略（per CONTEXT.md QTZ-01 decision）：
  添加 @GetMapping("/job") 显式字面路径处理器，返回相同视图 "monitor/job/job"。
  Spring MVC 优先匹配字面 /job 而非路径变量 /{jobId}，所以不再出现 400 错误。

已存在方法 view() (line 64-67)：
  @GetMapping()
  public String view() { return prefix + "/job"; }
  // 其中 prefix = "monitor/job"，所以返回 Thymeleaf 模板 templates/monitor/job/job.html
