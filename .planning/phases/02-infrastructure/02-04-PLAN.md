## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Environment (config) → @Bean init | Secret keys cross from application-*.yml into SDK init() calls — must not leak to logs |
| zgbas-framework → external HTTP services | 3 SDK facades make outbound HTTP to spt-auth/push/file servers (plaintext HTTP in dev — prod TLS is out-of-scope tech debt) |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-P2-log | Information Disclosure | ZgbasExternalBeansConfig logging secretKey/appCode | mitigate | DELETE the source's logger.info("url:{},secretKey:{},appCode:{}",...) line (FrameworkConfig.java:55). Verified SECRET_LOG == 0 in acceptance criteria. |
| T-P2-secret | Information Disclosure | Plaintext secrets in config | mitigate | Secrets are env-var placeholders in application-prod.yml (Plan 06, D-P2-13). This plan only reads them via env.getProperty — never hardcodes. |
| T-P2-collision | Tampering / DoS | ToolsJpaConfig vs ToolsMybatisConfig datasource bean race (Pitfall 1) | mitigate | Framework @Primary DataSource registered first → both spt-tools beans back off via @ConditionalOnMissingBean. Deterministic, no flaky ambiguity. |
| T-P2-http | Spoofing | External SDKs use plaintext HTTP URLs | accept | Production TLS termination at gateway is out-of-scope tech debt (RESEARCH §Security). Dev URLs are internal. |
</threat_model>
