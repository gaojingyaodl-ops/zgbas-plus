# Milestone v1.0: 单体化重构交付

**Status:** ✅ SHIPPED 2026-07-20
**Phases:** 1–7
**Total Plans:** 37

## Overview

zgbas-plus v1.0 是供应链核心管理平台 zgbas 从 Spring Cloud 微服务（4 服务）到 Maven 聚合单体（5 模块）的重构交付。7 个阶段按 foundation-first 递进，最终交付单进程启动即可跑通全部供应链业务（登录 → 核心业务 → 报表 → 定时任务）的单体应用，行为对齐旧系统 zgbas。

## Milestone Summary

**Key Decisions:**

| Decision | Rationale | Outcome |
|----------|-----------|---------|
| 参考框架 = RuoYi 单体（非 yudao） | yudao 需 Spring Boot 3 / JDK 17 / jakarta，与 JDK 1.8 + javax + Shiro + spt-tools 硬冲突 | ✓ Good |
| 认证保持外部 spt-auth（HTTP） | 符合用户 #7"外部 Bean 保持原方式"，改动最小 | ✓ Good |
| Spring Boot 2.5.9 / JDK 1.8 锁定不动 | 用户要求"尽量不变"；升级牵动 259 实体 javax→jakarta | ✓ Good |
| 双 ORM：单 DataSource + JpaTransactionManager @Primary | 两微服务本就连同一 schema | ✓ Good |
| spt-tools 内联顺序 core→(data,http,file)→(jpa,web,mybatis,shiro,aop) | core 是依赖根，阻塞其他；jpa 引用最广留后 | ✓ Good |
| D-P4-01 方案 A Feign 自回环 | 238 @FeignClient url=localhost:8080 同进程直调，无路由依赖 | ✓ Good |
| xxl-job → RuoYi quartz（整模块复制 spt-auth/auth-quartz） | 现成实现可降工作量 | ✓ Good |
| Phase 3 启动测试非 hermetic（Option 4） | 本地 export 密钥即可通过，CI/hermetic 化另立 task | ⚠️ Revisit |
