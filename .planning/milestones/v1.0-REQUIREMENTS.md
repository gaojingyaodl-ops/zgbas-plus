## v1 Requirements

### 骨架与打包 (BUILD)

- [x] **BUILD-01**: 搭建 5 模块 Maven 聚合单体 `zgbas-admin / zgbas-common / zgbas-framework / zgbas-quartz / zgbas-system`，模块间依赖关系正确 — *Validated in Phase 1*
- [x] **BUILD-02**: 使用 Maven `apache-maven-3.8.6` + `zg_settings.xml`（私服仓库）可干净构建 — *Validated in Phase 1*
- [x] **BUILD-03**: 锁定 JDK 1.8 + Spring Boot 2.5.9 版本基线，全模块一致 — *Validated in Phase 1*
- [x] **BUILD-04**: 单一 `@SpringBootApplication` 启动类，**单进程启动**即可加载全部模块 — *Validated in Phase 1*
- [x] **BUILD-05**: 仅 `zgbas-admin` 产出可执行 fat jar，弃旧 layout=ZIP 瘦 jar + libs 目录策略 — *Validated in Phase 1*

### 双 ORM 持久层 (PERSIST)

- [x] **PERSIST-01**: JPA 主力增删改查可用，迁移 259 实体 + 254 Dao（继承 `BaseDao`/`IdEntity` 体系）— *Validated in Phase 2（239 实体 + 240 Dao 审计链完整）*

- [x] **PERSIST-03**: 双 ORM 单 DataSource 共存（两服务本连同一 schema），`JpaTransactionManager` 设 `@Primary`，mybatis 复用同源 — *Validated in Phase 2*
- [x] **PERSIST-04**: 审计字段（`createdDate/updatedDate` + `@EntityListeners`）行为保留，`javax.persistence` 体系不变 — *Validated in Phase 2*

### spt-tools 内联 (INLINE)

### 外部服务集成 (EXT)

- [x] **EXT-01**: `AuthOpenFacade`(auth-sdk) 保持 `@Bean init(secretKey,appCode,url)` HTTP 注入，调外部 spt-auth — *Validated in Phase 2*
- [x] **EXT-02**: `PushClientHttp`(spt-push-sdk)、`FileRemote`(spt-file-sdk) 保持原 HTTP 注入方式 — *Validated in Phase 2*
- [x] **EXT-03**: `CfcaSignClient`(spt-sign-client) OpenFeign 保持，`@EnableFeignClients(basePackages=...,"com.spt.sign.client.remote")` 保留 — *Validated in Phase 2（Feign 收窄）*

### 认证与首页 (AUTH)

- [x] **AUTH-01**: 登录接口与流程照搬旧项目（Shiro session+cookie，非 JWT）— *Validated in Phase 3（Login/Index/SSO Controller + Shiro Realm/Service/Util）*

### 基础设施收敛 (INFRA)

## Out of Scope

| Feature | Reason |
|---------|--------|
| 合并 spt-auth 进单体 | 认证保持外部 HTTP（用户决策 #7），spt-auth 作共享基础设施 |
| Spring Boot 3 / JDK 17 升级 | javax→jakarta 影响 259 实体 + 254 Dao，与 JDK 1.8 硬约束冲突 |
| yudao 框架 | 需 Spring Boot 3 / JDK 17 / jakarta，技术栈不兼容 |
| 报表物理分页改造 | 旧自研 PageHelper 是结果包装器，本期仅保行为等价 |
| 补 createBy/updateBy/逻辑删除字段 | 涉及 259 表结构变更，本期不做 |
| basWx 小程序迁入 | 第一阶段不迁，v2（用户决策 #14） |
