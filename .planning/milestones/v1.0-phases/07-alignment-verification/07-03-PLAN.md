<action>
**15 REVIEW-flagged sys_job operator review**

此任务需人工逐条审查 15 个 REVIEW 标记的 sys_job 行，对每条做出决策：

1. 打印/展示 07-RESEARCH.md §3 的 15 行审查表（job_id / 名称 / invoke_target / REVIEW 原因 / 建议操作）
2. 对每条，请操作者选择：
   - **KEEP** — 保持原样（忠实于源，如 id=141 极端 cron 但 PAUSED）
   - **MODIFY** — 修改 args/参数（如 id=142 refreshProfitData 需要真实 approveNo）
   - **PAUSE** — 暂停任务（status='1'，不触发直到参数确认）
3. 收集全部 15 条决策
4. 对选择 MODIFY 的行：更新 `sys_job_data.sql` 中对应 INSERT 的 invoke_target 字段，使用操作者提供的新参数值
5. 对选择 PAUSE 的行：更新 `sys_job_data.sql` 中对应 INSERT 的 `status` 字段从 '0' 改为 '1'（PAUSED）
6. 重应用 SQL 到 dev DB：
   `mysql -u sptbaspduser -p sptbasdb_pd < zgbas-quartz/src/main/resources/sql/sys_job_data.sql`
   （注意：INSERT 会因主键冲突失败 — 需先 DELETE 再 INSERT，或使用 UPDATE 语句只修改变更的行）
7. 验证修改后 `schedulerLoadAllJobs_proof` 仍通过（重新启动应用确认 53 行加载）

checkpoint:human-verify — 每条决策需操作者确认。
</action>

<acceptance_criteria>
- 15 条 REVIEW sys_job 全部完成 operator review（每条有 KEEP/MODIFY/PAUSE 决策记录）
- 选择 MODIFY 的行在 `sys_job_data.sql` 中 invoke_target 已更新
- 选择 PAUSE 的行在 `sys_job_data.sql` 中 status 改为 '1'
- 修改后的 SQL 已重应用到 dev DB `sptbasdb_pd`
- 应用重启后 `schedulerLoadAllJobs_proof` 通过（53 行加载）
- `grep -c 'REVIEW' sys_job_data.sql` 返回 0（REVIEW 标记全部清除或替换为决策记录）
</acceptance_criteria>
</task>
