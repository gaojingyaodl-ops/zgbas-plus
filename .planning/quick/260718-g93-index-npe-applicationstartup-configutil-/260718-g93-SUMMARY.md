# Quick Task 260718-g93: Fix /index NPE — init auth-sdk ConfigUtil cache on startup

## Summary

Post-login `GET /index` NPE'd at `ConfigUtil.getValue:52` because the monolith startup listener `ApplicationStartup` initialized 11 sibling auth-sdk caches but omitted `ConfigUtil.init()`, leaving `ConfigUtil.configCache` (a static Guava `LoadingCache`) null until explicitly initialized. The init lived in the old web service and was lost during the web→monolith migration. Fixed by adding `ConfigUtil.init()` alongside the existing `DictUtil.init(appCode)` call in `ApplicationStartup.onApplicationEvent`, plus the corresponding import.
